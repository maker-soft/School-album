# НОВЫЕ ГРАНИ — полноценный режим

Этот комплект работает как статический сайт на GitHub Pages, а данные, авторизация, фотографии и базовая статистика хранятся в Supabase.

## 1. Создайте проект Supabase

1. Откройте Supabase Dashboard и создайте новый проект.
2. Откройте **SQL Editor**.
3. Скопируйте весь файл `supabase/setup.sql` и выполните его.

Скрипт создаёт:

- таблицу контента;
- таблицу администраторов;
- таблицу событий статистики;
- публичный Storage bucket `site-images`;
- Row Level Security-политики;
- функцию сводной статистики;
- исходный тестовый контент.

## 2. Создайте администратора

1. Откройте **Authentication → Users**.
2. Добавьте пользователя с вашим email и паролем либо отправьте себе приглашение.
3. Убедитесь, что пользователь подтверждён.
4. Скопируйте его UUID.
5. Откройте `supabase/add-admin.sql`, замените тестовый UUID и выполните запрос в SQL Editor.

## 3. Заполните config.js

Откройте `config.js` и замените:

```js
SUPABASE_URL: "https://YOUR_PROJECT.supabase.co",
SUPABASE_PUBLISHABLE_KEY: "YOUR_PUBLISHABLE_OR_ANON_KEY"
```

Значения находятся в Supabase Dashboard в **Settings → API Keys / Data API**.

Используйте только **Publishable key** или legacy **anon key**. Никогда не вставляйте в сайт Secret key или `service_role` key.

Необязательно укажите ID Яндекс Метрики:

```js
YANDEX_METRIKA_ID: "12345678"
```

## 4. Разместите на GitHub Pages

1. Создайте новый GitHub-репозиторий.
2. Загрузите в корень репозитория содержимое этой папки, а не внешнюю папку целиком.
3. Откройте **Settings → Pages**.
4. В **Build and deployment** выберите **Deploy from a branch**.
5. Ветка: `main`; папка: `/(root)`.
6. Сохраните настройки.

Публичный сайт:

```text
https://ИМЯ.github.io/РЕПОЗИТОРИЙ/
```

Админка:

```text
https://ИМЯ.github.io/РЕПОЗИТОРИЙ/admin/
```

## 5. Проверьте работу

1. Откройте публичный сайт.
2. Откройте `/admin/` и войдите под созданным email и паролем.
3. Измените стоимость любого альбома и нажмите **Сохранить**.
4. Обновите публичный сайт в другом браузере или на телефоне.
5. Загрузите тестовую фотографию и снова сохраните изменения.
6. Выполните несколько действий на сайте и проверьте вкладку **Обзор** в админке.

## Что хранится где

- GitHub Pages: HTML, CSS и JavaScript.
- Supabase Database: цены, тексты, FAQ, ссылки на изображения.
- Supabase Storage: загруженные фотографии.
- Supabase Auth: email и пароль администратора.
- `site_events`: базовые просмотры и клики.
- Яндекс Метрика: расширенная аналитика, источники трафика, устройства, география и визиты.

## Ограничения

- Базовая статистика сайта предназначена для оперативного контроля, а не для сложной веб-аналитики.
- Удаление фотографии из галереи убирает её с сайта, но сам файл остаётся в Storage. Ненужные файлы можно периодически удалять через Supabase Storage.
- Перед загрузкой рекомендуется переводить фотографии в WebP, ширина 1600–2200 px, вес примерно 200–700 КБ.
